# GymJDBC-LR5

Лабораторная работа №5 по JDBC.  
Проект представляет собой настольное Java-приложение с графическим интерфейсом для работы с базой данных тренажёрного зала в PostgreSQL.

## Описание проекта

В рамках работы реализована однотабличная база данных по предметной области **тренажёрный зал**.  
Основная сущность системы — **клиент фитнес-клуба**, так как именно регистрация, хранение данных о клиентах и управление ими являются базовой частью работы любой подобной системы.

Приложение позволяет:

- создавать базу данных;
- инициализировать структуру базы;
- подключаться к созданной базе;
- входить в систему под разными пользователями;
- просматривать список клиентов;
- искать клиентов по фамилии;
- добавлять, изменять и удалять записи;
- очищать таблицу;
- создавать и удалять пользователей БД;
- удалять всю базу данных.

## Предметная область

В проекте используется таблица клиентов тренажёрного зала:

- `client_id` — идентификатор клиента;
- `last_name` — фамилия;
- `first_name` — имя;
- `phone` — телефон;
- `email` — электронная почта;
- `membership_type` — тип абонемента;
- `status` — статус клиента;
- `registration_date` — дата регистрации.

## Используемые технологии

- **Java**
- **Swing** — графический интерфейс
- **JDBC**
- **PostgreSQL**
- **PL/pgSQL**
- **IntelliJ IDEA**

## Функциональные возможности

### Для всех пользователей
- просмотр всех клиентов;
- поиск клиентов по фамилии;
- обновление отображаемого списка.

### Для администратора
- добавление клиента;
- редактирование данных клиента;
- удаление клиентов по фамилии;
- очистка таблицы;
- создание новых пользователей БД;
- удаление пользователей БД;
- удаление всей базы данных.

### Для гостя
Гостевой режим работает в формате **только чтение**:

- можно просматривать данные;
- можно выполнять поиск;
- нельзя добавлять, изменять или удалять записи;
- нельзя создавать или удалять пользователей;
- нельзя удалять базу данных.

## Ролевая модель

В проекте используются две основные роли:

- `admin_role` — полный доступ к данным и административным операциям;
- `guest_role` — только просмотр и поиск.

Также может использоваться служебный пользователь для создания и инициализации базы данных.

## Особенности реализации

Одно из требований лабораторной работы — использование **хранимых процедур и функций** при работе с базой данных.  
Поэтому в Java-коде все основные действия выполняются через вызовы процедур и функций PostgreSQL.

В проекте используются процедуры и функции для:

- создания базы данных;
- удаления базы данных;
- инициализации структуры;
- добавления клиента;
- обновления клиента;
- удаления клиентов по фамилии;
- очистки таблицы;
- получения всех клиентов;
- поиска клиентов;
- проверки прав администратора;
- создания пользователей БД;
- удаления пользователей БД.
